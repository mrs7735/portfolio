Memory Forensics: Hunting Malware in RAM with Volatility

DATE:2025.01.28
READ:12 MIN READ
AUTHOR:CYBER_OPERATIVE
Memory ForensicsVolatilityDFIRIncident ResponseMalware

When malware runs, it leaves traces in memory that disappear on reboot — but during an incident, RAM is a goldmine. Memory forensics with Volatility can reveal running processes, injected code, network connections, and credentials that never touched disk.

1. Acquiring a Memory Image

# Windows — must run as Administrator
winpmem_mini_x64.exe memdump.raw
# Linux
sudo dd if=/proc/kcore of=memdump.raw
# Virtualised environments — suspend VM and copy .vmem file

2. Volatility 3 Basics

# Process list
vol -f memdump.raw windows.pslist
vol -f memdump.raw windows.pstree  # parent-child relationships
# Network connections
vol -f memdump.raw windows.netstat
# DLL list for a process
vol -f memdump.raw windows.dlllist --pid 1337

3. Finding Suspicious Processes

Know the normal Windows process tree. Anything spawning from an unusual parent or running from temp directories is suspicious.

# Check for process hollowing / injection
vol -f memdump.raw windows.malfind
# Compare PID list vs running processes (detect DKOM hiding)
vol -f memdump.raw windows.pslist > pslist.txt
vol -f memdump.raw windows.psscan > psscan.txt
diff pslist.txt psscan.txt  # hidden processes appear only in psscan

4. Extracting Artefacts

# Dump a suspicious process
vol -f memdump.raw windows.dumpfiles --pid 1337
# Extract registry hives from memory
vol -f memdump.raw windows.registry.hivelist
vol -f memdump.raw windows.registry.printkey --key "SOFTWARE\Microsoft\Windows\CurrentVersion\Run"
# Recover command history
vol -f memdump.raw windows.cmdline
vol -f memdump.raw windows.consoles  # actual typed commands

5. Credential Extraction from Memory

# LSASS process memory — contains plaintext creds on older Windows
vol -f memdump.raw windows.hashdump
# Mimikatz-equivalent through Volatility
vol -f memdump.raw windows.lsadump
